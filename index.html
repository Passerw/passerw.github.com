安装说明

Posted on 2020-03-24 Edited on 2019-11-25

  1. ##安装环境

    1. nodejs环境

      • node 10.16.3 –nodejs环境
      • npm 6.9.0 –node自带npm
      • nrm 1.2.1 –切换npm镜像常用命令nrm ls , nrm use
      • ionic 4.6.0 –ionic命令行工具(升级版本对开发有影响)
      • cordova 8.0.0 –cordova命令行工具

    2. android环境

      • sdk 26.1.1+检查sdk\tools\templates\gradle必须存在
      • 解压sdk压缩包
      • –配置环境变量
        • 增加名称:
          • ANDROID_HOME 值： sdk目录
        • path中配置
          • %ANDROID_HOME%\platform-tools
          • %ANDROID_HOME%\tools
      • —-cmd执行adb命令是否可用测试sdk配置成功

    3. ios环境

      • 安装xcode,并启动提示安装依赖，确定，成功启动后环境就好了
        ##安装项目依赖

    4. 第一次，删除node_models

    5. 安装依赖

      1. nrm use taobao
      2. cd到项目根目录
      3. SET SASS_BINARY_PATH=%cd%/build/win32-x64-64_binding.node && npm i

    6. 添加平台

      • ionic cordova platform add android@8.0.0
      • ionic cordova platform add ios@5.5.0
        ##开发

    7. ionic g

    • 生成组件，执行后选择组件类型，输入组件名称，命名规范单词首字母大写
      ionic cordova run android -lc

##问题记录

  1. no such file or directory, scandir ‘*\node_modules\node-sass\vendor’
    • 因为没有成功安装bind文件
    • 解决
      • SET SASS_BINARY_PATH=%cd%/build/win32-x64-64_binding.node && npm rebuild node-sass
      • 查看node-modeles下的node-sass有了vendor/win32-x64-64目录
      • 将build中的win32-x64-64_binding.node复制到vendor/win32-x64-64目录，改名binding.node
  2. -lc 运行到手机的app加载报错ERR_ClEARTEXT_NOT_PERMITTED
    • 因为手机系统9.0的安全限制
    • 解决
      • 在项目resources下android下xml中network_security_config.xml配置请求地址
  3. 组件加载不进来，执行命令ionic cordova prepare android
